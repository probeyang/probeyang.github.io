    nonlocal的使用

    什么是nonlocal（非局部语句）

    nonlocal是Python 3.x中新引入的特性，可以让你给外层但非全局作用域中的变量赋值。官方文档中的说法是，非局部语句可以让所列的标识符（identifier）指向最近的嵌套作用域（enclosing scope）中已经绑定过的变量，全局变量除外。

    python引用变量的顺序

    当前作用域局部变量->外层作用域变量->当前模块中的全局变量->python内置变量

    示例

    class Demo(object):
    test = None

    def __init__(self):
        field = "dddd"

    def sigma(self):
        def do_local():
            field = "local field"  # 此函数定义了另外的一个field字符串变量，并且生命周期只在此函数内。此处的field和外层的field是两个变量，如果写出field = field + “local field” 会报错

        def do_nonlocal():
            nonlocal  field  # 使用外层的field变量
            field = "nonlocal field"

        def do_global():
            global field
            field = "global field"

        field = "test field"
        do_local()
        print("After local assignmane:", field)
        do_nonlocal()
        print("After nonlocal assignment:", field)
        do_global()
        print("After global assignment:", field)

demo = Demo()
demo.sigma()
print("In global field:", field)

    返回结果：

    /Library/Frameworks/Python.framework/Versions/3.5/bin/python3.5 /Users/probeyang/study/python/test/test7.py
After local assignmane: test field
After nonlocal assignment: nonlocal field
After global assignment: nonlocal field
In global field: global field

Process finished with exit code 0

    解释

    在do_nonlocal函数的中添加了nonlocal field语句，这个语句是告诉Python解释器在碰到为field赋值的语句时，应该向外层作用域的变量赋值，而不是声明一个重名的新变量。这样，两个函数的打印结果就一致了。

    nonlocal的用法和global非常类似，只是前者针对的是外层函数作用域的变量，后者针对的则是全局作用域的变量。

    另外注意如果参数是dict类型的数据，则不需要nonlocal语句，因为这种类型的参数插入并不是赋值操作，而是方法调用（method call），示例：

    >>> def outside():
        d = {"outside": 1}
        def inside():
            d["inside"] = 2
            print(d)
        inside()
        print(d)

>>> outside()
{'inside': 2, 'outside': 1}
{'inside': 2, 'outside': 1}
