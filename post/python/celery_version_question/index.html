查了资料说是因为Celery版本4.1.0的问题，默认情况下我们只在项目的requirement.txt文件中设置了Celery的版本：

celery==4.1.0

而没有设置Celery的依赖包的版本： kombu 所以pip会自动获取kombu的最新版本 kombu 4.2.0。然而这个版本和Celery的4.1.0有兼容问题，就要报上面的错误，所以我们需要按照网上文献中说的讲kombu的版本：

kombu 4.1.0

这样问题就解决了。 另外，一般Celery的key不存在的问题只需要重启Celery就可以解决的。
