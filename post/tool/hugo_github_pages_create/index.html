使用hugo大家github pages站点

web
次阅读

文章目录

    • 环境介绍
    • 搭建步骤
    • 参考文献

【注意】最后更新于 September 24, 2018，文中内容可能已过时，请谨慎使用。

本文主要介绍如何使用Hugo在Github pages搭建免费的个人博客，关于Hugo可在其官网上进一步了解。

环境介绍

Mac OSX 10.11.1 项目根路径 ~/study/page_github

搭建步骤

  • 在Github创建github pages仓库，这里不做赘述
  • 安装Hugo
brew install hugo
  • 使用hugo创建站点
cd /study/page_github
hugo new site hugo.sanlt.com
cd hugo.sanlt.com
  • 使用主题

主题比较多，这里使用对中文支持比较好且git上star比较多的一个主题even：（确保当前目录在hugo.sanlt.com这里）

git clone --recursive git@github.com:olOwOlo/hugo-theme-even.git themes/even

上面命令的意思是在themes文件夹下面拉取并生成一个even的主题。

  • 新建md文章
hugo new about.md

注意，这里写文章需要注意每个文章开头需要添加一段以下内容的东西

在content文件夹下新建post文件夹，在post中创建.md文件即可，但是要注意的是文件要包含一下特殊的内容，如下所示

---
title: 新文章
slug: "hello"
date: "2018-02-01"
tags: ["web","python"]
categories: ["python"]
mathjax: true
---

在文件中加入这些内容即可，事实上，这些就是这篇文章的基本信息：标题、url、日期、标签和分类。内容可以没有上面这么多，具体参数含义可以参考网络上文献。

更注意：图片地址默认在static文件夹下面，我一般习惯在static文件夹下新建一个img文件夹放所有博客内容图片，路径写法如下：

/img/sdfisre44efdg43wet345.jpg

  • 启动服务
 hugo server -t even --buildDrafts
  • 访问服务

浏览器访问 http://localhost:1313/ 看到如下图页面，说明本地搭建博客已经没问题了。

我这里是加了不少页面内容了的。

接下来讲解怎么将网站内容放到github pages。

  • 生成public文件

这一步网上资料说的比较含糊，一笔带过，这里我明确一下，是需要在根目录运行一个命令的，而这个命令就是：hugo

hugo

这时会在根目录生成一个public文件夹，将public文件夹放到github pages对应的github地址并提交到远端即可。

  • 插件安装

有很多插件需要安装：评论、网站阅读量等等。这个主题自带了busuanzi的阅读量服务，只需要修改配置即可。

  • Favicon 为了定制 favicon，你需要在站点根目录的 static 文件夹下放置下述所有文件，这将覆盖 themes/even/static/ 文件夹下的那些文件。

android-chrome-192x192.png android-chrome-512x512.png apple-touch-icon.png browserconfig.xml favicon.ico favicon-16x16.png favicon-32x32.png manifest.json mstile-150x150.png safari-pinned-tab.svg favicon generator (Google) 能够帮助你生成这些文件。
