SQLAlchemy报错ImportError No module named 'mysql'解决

问题描述

在使用SQLAlchemy的使用，实例化数据库引擎语句：

1

create_engine('mysql+mysqlconnector://root:123456@localhost:3306/blogs’)

报错：

1

ImportError: No module named 'mysql'

问题解决

查询资料后发现，其实是创建引擎时里面的用法不对导致的， > If you’re having issues compiling the binary extension, or on a platform where you cant, you can try using the pure python PyMySQL bindings. > Simply pip install pymysql and switch your SQLAlchemy URI to start like this: > SQLALCHEMY_DATABASE_URI = ‘mysql+pymysql://…..’

我们可以改创建语句为：

1

engine = create_engine('mysql+pymysql://root:123456@localhost:3306/blogs')

这个语句注意要本地安装有pymysql：pip3 install pymysql 或者：

1

engine = create_engine('mysql://root:123456@localhost:3306/blogs')

这两种改法都是可以取出数据来的。

问题延伸

为什么字符串是那样的？我们来研究一下： 实际上是

1

数据库+数据库驱动

这样的格式的。数据库+数据库驱动，只要安装了相应的数据库和驱动都能使用。

1、create_engine(‘mysql+mysqlconnector://root:password@localhost:3306/test’) 这个mysqlconnector是MySQL自带的数据库驱动程序

2、create_engine(‘mysql+mysqldb://root:password@localhost:3306/test’) 使用mysqldb驱动连接数据库，如果连接不上可以换localhost为127.0.0.1试试。

3、create_engine(‘mysql://root:password@localhost:3306/test’) 这个没有驱动器的情况下，我尝试过也没有问题，如果连接不上可以换localhost为127.0.0.1试试。

还有就是要不要端口都是没有问题的，3306是数据库初始化时给的值，所以也就是默认值，自己也可以另外设置。
