简述

php-cs-fixer是比较专业的PHP代码格式化工具，比phpstorm自带的格式化功能更多，可以自定义很多参数，这样能满足我们特别的代码规范。目前我们的代码规范也没有太多特殊性，按照行业规范PSR标准即可。参考几个代码格式化工具扩展，php-cs-fixer是比较好的选择。

安装

  1. 从https://github.com/FriendsOfPHP/PHP-CS-Fixer/blob/2.12/README.rst#installation下载php-cs-fixer.phar 文件，注意使用v2版本，因为目前的PHP版本都是5.6以后的，而v1版本是支持PHP5.6以前版本的。

  2. 将文件放到PHP的安装目录里面，跟PHP.exe在同一级。

  3. 可以将php-cs-fixer-v2.phar修改为php-cs-fixer.phar

配置phpstorm

  1. file-setting-tools-external tools, 在里面添加外部扩展工具，点击绿色的+号

  2. program中选择PHP的目录。

  3. arguments输入：php-cs-fixer.phar路径 fix –rules=@PSR1,@PSR2 –verbose $FileDir$$FileName$

  4. working directory输入：$ProjectFileDir$

点击OK，以及apply，完成配置。即可完成php-cs-fixer的配置。具体配置如图：

尝试试用

  1. 可以选择顶部tools-external tools-php-cs-fixer即可运行格式化工具

  2. 或者在代码中使用右键找到external tools也可以使用格式化工具

输出类似：

配置快捷键

觉得很麻烦？还可以添加快捷键来直接使用格式化功能：

  1. file-setting-keymap-external tools找到php-cs-fixer，双击或者右键add keyboard shortcut

  2. 直接输入想要设置的快捷键，注意不要跟其他快捷键冲突即可。

总结

php-cs-fixer相对比较专业的格式化工具，技术社区也比较成熟，能很好找到各种资料配置。并且它比phpstorm自带的专业的多，更符合代码审核规范，提高代码质量，配置还很简单。用惯了此功能后，再看其他代码将会显得差距大很多。
