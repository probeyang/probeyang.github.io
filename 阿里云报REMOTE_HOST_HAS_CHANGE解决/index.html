    阿里云服务器报REMOTE HOST IDENTIFICATION HAS CHANGED错的处理方法

    阿里云服务器登录的时候报类似下面的错误：

    [ddddd@prdesedeMacBook-Pro ~ ]$ ssh root@126.123.123.119
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@    WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!     @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!
Someone could be eavesdropping on you right now (man-in-the-middle attack)!
It is also possible that a host key has just been changed.
The fingerprint for the ECDSA key sent by the remote host is
SHA256:U8iyiutuuLRmHj8H7K/asdfHOzxdsadsFRkrsdfsd2IrGnNLrcH40.
Please contact your system administrator.
Add correct host key in /Users/ddddd/.ssh/known_hosts to get rid of this message.
Offending ECDSA key in /Users/ddddd/.ssh/known_hosts:8
ECDSA host key for 126.123.123.119 has changed and you have requested strict checking.
Host key verification failed.

    怎么解决呢？

    先说一下问题产生的原因：

    之前登录了这台服务器，然后保存了跟服务器之间的通信秘钥。过了一段时间这台机器进行了一系列的维护重启，对其中的一些服务做了重新部署。我怀疑是这里出了问题。

    解决办法：

    查了一下网上的资料，其实有比较简单的解决方式，

    在命令行执行keygen重置命令：

    ssh-keygen -R 126.123.123.119

    实际操作示例：

    [ddddd@prdesedeMacBook-Pro ~ ]$ ssh-keygen -R 126.123.123.119
# Host 126.123.123.119 found: line 8
/Users/ddddd/.ssh/known_hosts updated.
Original contents retained as /Users/ddddd/.ssh/known_hosts.old

    这个命令就是重置本地客户端的秘钥。

    再重新执行登录

    ssh root@126.123.123.119

    就可以重新建立秘钥和服务器端进行连接了。
